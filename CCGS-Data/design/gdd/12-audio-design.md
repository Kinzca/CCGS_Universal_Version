# Game Design Document: 音频设计

> **系统：** Audio Design — SFX, Music & Sonic Feedback  
> **版本：** 1.0  
> **设计：** Game Designer

---

## 1. 音频设计原则

### 1.1 核心理念

| 原则 | 说明 |
| :--- | :--- |
| **操作有回声** | 每一个玩家操作（成功/失败）都必须有对应的音效反馈，沉默 = 系统无响应 |
| **空间可听性** | 音效应辅助空间感知，帮助玩家理解 3D 结构中的位置关系 |
| **认知负荷控制** | 音效数量和强度不应造成听觉疲劳，频繁操作的音效必须克制 |
| **风格统一** | 所有音效共享统一的音色调性：简洁、几何感、略带空灵质感 |

### 1.2 音频层次模型

```
Layer 3: 环境音 (Ambient)     — 持续低频，营造氛围
Layer 2: 音乐 (Music)        — 自适应背景音乐
Layer 1: 系统音效 (System SFX) — UI、过渡、成就
Layer 0: 操作音效 (Action SFX) — 最高优先级，直接反馈玩家行为
```

> **冲突规则**：当 Layer 0 触发时，Layer 2/3 自动降低音量（ducking），确保操作反馈清晰可辨。

## 2. 操作音效 (Action SFX)

### 2.1 旋转系统

| 事件 | 音效描述 | 时长 | 音量 | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **旋转成功** | 清脆的机械齿轮啮合声 + 轻微的"咔哒"落位声 | ~0.3s | 中 | 随旋转动画同步播放 |
| **旋转失败（安全锁定）** | 低沉的金属碰撞 + 短促的"嗡"振动声 | ~0.2s | 中偏低 | 与微旋转弹回动画同步 |
| **QE 场景旋转** | 轻柔的风声/空间旋转声 | ~0.35s | 低 | 随 DOTween 场景旋转同步 |

### 2.2 移动系统

| 事件 | 音效描述 | 时长 | 音量 | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **平面移动** | 轻柔的脚步声 / 滑动声 | ~0.2s | 低 | 每步一次，频率克制 |
| **楼梯攀爬** | 略重的脚步声 + 微弱的高度变化风声 | ~0.3s | 中偏低 | 区分上下楼 |
| **移动失败（导航阻塞）** | 极轻的碰壁声 / "tock" | ~0.15s | 低 | 与角色前倾弹回同步 |

### 2.3 窗口过渡

| 事件 | 音效描述 | 时长 | 音量 | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **退场掉落** | 石块崩落声（由远及近的低频）| ~0.5s | 中 | 多方块错落，音量叠加控制 |
| **入场掉落** | 石块落定声 + 轻微回弹 | ~0.45s | 中 | 错落有致的"咚咚"声 |
| **窗口推进成功** | 上升感的音效（如风铃递增音阶）| ~0.8s | 中 | 与相机运镜同步 |

### 2.4 撤销 (Undo)

| 事件 | 音效描述 | 时长 | 音量 | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **撤销旋转** | 旋转成功音效的反向播放（倒带感）| ~0.3s | 中偏低 | 暗示"时间回溯" |
| **撤销移动** | 轻柔的滑动回退声 | ~0.2s | 低 | |

## 3. 系统音效 (System SFX)

| 事件 | 音效描述 | 时长 | 音量 |
| :--- | :--- | :--- | :--- |
| **到达出口** | 成就达成的和弦（升调，温暖感）| ~1.0s | 中偏高 |
| **首次进入塔** | 空旷的空间回响 + 轻微的风声 | ~1.5s | 中 |
| **DDA 难度降级** | 无显式音效（避免暗示"你太菜了"）| — | — |
| **DDA 难度提升** | 无显式音效 | — | — |

> **DDA 静默原则**：难度调整对玩家完全透明，不通过音效暴露 DDA 的存在，维护玩家的自主感和控制感。

## 4. 环境音 (Ambient)

### 4.1 设计方向

| 维度 | 设计决策 |
| :--- | :--- |
| 基调 | 空灵、几何、微机械感——与视觉风格匹配的"抽象空间" |
| 动态性 | 随攀升高度变化：低层→沉稳厚重；高层→空旷高远 |
| 循环策略 | 无缝循环的 ambient pad，避免明显的循环接缝 |

### 4.2 高度分层

| 高度区间 | 环境音特征 |
| :--- | :--- |
| 0–10 层 | 低频嗡鸣 + 微弱的石材质感 |
| 10–30 层 | 增加风声分量，低频逐渐减弱 |
| 30+ 层 | 高频空灵泛音主导，极简质感 |

## 5. 音乐 (Music)

### 5.1 设计方向

| 维度 | 设计决策 |
| :--- | :--- |
| 风格 | 极简电子 / Ambient Electronica |
| 自适应 | 音乐层次随玩家状态（心流/焦虑/无聊）动态增减 |
| 节奏 | 非节拍驱动——以 pad 和 texture 为主，避免与操作节奏冲突 |

### 5.2 DDA 联动（远期规划）

当 DDA 系统正式上线后，音乐系统可与心流状态联动：

| 心流状态 | 音乐响应 |
| :--- | :--- |
| 心流通道内 (CSR ≈ 1.04) | 完整音乐层，和谐稳定 |
| 焦虑区 (CSR > 1.2) | 减少音乐层次，降低节奏密度 |
| 无聊区 (CSR < 0.8) | 增加音乐层次/紧张感，暗示挑战 |

> **当前状态**：音乐系统尚未实现，此为远期规划方向。

## 6. 技术实现

### 6.1 音频管理架构

| 组件 | 职责 |
| :--- | :--- |
| **AudioManager** | 全局音频管理，音量控制，ducking 策略 |
| **SfxPlayer** | 即时音效播放器，支持 OneShot 和对象池复用 |
| **AmbientController** | 环境音分层控制，高度联动 |
| **MusicController** | 自适应音乐分层控制（远期）|

### 6.2 性能约束

| 约束 | 说明 |
| :--- | :--- |
| 同时播放上限 | 操作音效最多 3 路并发，避免音频混叠 |
| 音效复用 | 高频音效（移动脚步）使用对象池 |
| 内存预算 | 所有音效资源总大小不超过 10MB |

> **当前状态**：音频系统尚未实现，本文档为设计规范，待工程阶段逐步落地。
