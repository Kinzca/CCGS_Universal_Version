## 13.1 本地化架构概述

交错空间采用**配置表驱动**的本地化架构，所有用户可见文本通过 Luban 配置表管理，运行时按语言代码加载对应列。

### 架构层次

```
应用层      UI 组件通过 Key 引用文本
              ↓
管理层      LocalizationManager 按语言加载配置
              ↓
数据层      Luban 配置表（多语言列）
              ↓
资源层      字体资产 / 本地化图片 / 音频
```

---

## 13.2 本地化就绪检查清单

> 每个阶段完成时对照检查，确保本地化就绪度持续达标。

### 文本层

- [ ] 所有用户可见文本通过配置表 Key 引用，无硬编码字符串
- [ ] Key 命名遵循 `[模块]_[功能]_[描述]` 规范（如 `tower_hint_rotateGuide`）
- [ ] 配置表支持多语言列（`zh-CN` / `en-US` / ...）
- [ ] 占位符使用参数化格式（如 `{0} 层` 而非字符串拼接）

### UI 层

- [ ] UI 容器使用自适应布局，不硬编码文本区域尺寸
- [ ] 文本组件使用 TextMeshPro，支持多语言字符集
- [ ] 预留文本长度弹性（英文可能比中文长 30%~50%）
- [ ] 图标/按钮中无内嵌文字（或有替换方案）

### 资源层

- [ ] 字体资产按语言分组，按需加载避免内存膨胀
- [ ] 含文字的图片资源有多语言版本或无文字替代方案
- [ ] 音频旁白（如有）有多语言版本或字幕方案

### 文化层

- [ ] 颜色使用无文化冲突（红/绿在不同文化中含义不同）
- [ ] 日期/数字/货币格式可配置
- [ ] 无特定文化敏感内容
