## 关键依赖链说明

### 核心三角：03 ↔ 04 ↔ 05（塔 ↔ 生成 ↔ 方块）

这三章构成最紧密的耦合关系：
- **03 塔系统**定义窗口语义模型 → **04 生成**基于此模型生成内容 → **05 方块**定义生成可用的方块类型
- 修改任一章节时，必须逐条检查另外两章是否受影响
- 特别注意：`TowerWindowGenerationResult` 真源定义在 03，但生成逻辑在 04，方块规则在 05

### 数值闭环：04 ↔ 06 ↔ 09（生成 ↔ DDA ↔ 配置）

- **06 DDA** 的 CSR 方程参数 → 影响 **04 生成** 的打乱步数 M
- **09 配置** 的 Luban 三表结构 → 承载 04 和 06 的所有可调参数
- 修改 DDA 参数时必须验证配置表字段是否对齐

### 输入链：02 → 03 → 12（操控 → 塔系统 → 音频）

- **02 操控** 的输入映射变更 → 可能影响 **03 塔系统** 的旋转/移动触发
- 操作反馈音效在 **12 音频** 中定义 → 新增操作类型时需同步

### 学术链：11 → 06 → 04（学术 → DDA → 生成）

- **11 学术** 的理论模型变更 → 影响 **06 DDA** 的算法设计
- DDA 约束变更 → 影响 **04 生成** 的验收准则

## Phase 4 使用指引

当 Phase 4 需要更新 GDD 时，按以下步骤操作：

1. 确定本次变更涉及的**主章节**
2. 查阅上方矩阵，找到所有 ● 强依赖的关联章节
3. 逐章检查：关联章节中是否存在与本次变更**语义冲突**的描述
4. 若存在冲突 → 同步更新关联章节
5. 若无冲突但描述过时 → 标记为"待下次更新时同步"
6. 在 Changelog 中记录本次跨章节同步情况
