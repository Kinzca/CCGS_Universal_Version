# 14. UI 与交互设计 (UI & UX Design)

> **修订版本**：1.0
> **日期**：2026-04-23
> **作者**：Game Designer (@game-designer)
> **状态**：Draft

## 1. 概述
本文档定义了《交错空间》(InterwovenSpace) 的用户界面（UI）与交互设计（UX）规范。遵循极简几何风格与认知负荷理论（CLT），以降低玩家认知成本为核心目标。

## 2. 视觉场景基调 (Scene & Visual Foundation)
* **构图范式**：全屏采用 16:9 左右分栏构图。左侧为 3D 渲染区域，右侧为 2D 磨砂玻璃（Glassmorphism）UI 交互区域。
* **3D 场景呈现（纯粹悬浮流）**：
  * 主场景没有物理地面、底座或云雾遮挡。
  * 左侧的 3D 物件（如主菜单的解谜高塔、设置界面的方块齿轮）完全悬浮于渐变虚空的天空盒中。
  * 3D 塔的光影沿用 `07-rendering-visual.md` 中的「等距三面着色」无光照材质，配合后处理泛光（Bloom）。
  * 天空盒颜色随游戏模式（或塔的高度）动态流转。

## 3. 界面交互逻辑 (Interaction Logic)
* **主菜单切换机制**：通过显性的 UI 导航按钮（或后台安全区内的左右滑动）切换不同的游戏模式（如每日挑战、无尽高塔）。左侧的 3D 模型随模式无缝转场。
* **设置界面「锚点平滑滚动」机制（核心 UX）**：
  * 右侧设置区**不采用硬切页**，而是一个统一的、可上下滚动的长列表。
  * 顶部的页签（通用、控制、画面、音频）作为**快速跳转锚点 (Jump Anchors)**。
  * **行为表现**：玩家点击“控制”时，列表平滑自动下拉，将控制区域的头部对齐到视口顶端；玩家手动向下滑动列表时，顶部的页签高亮状态会根据当前视口所在区域自动点亮对应的文字。

### 3.1 主菜单与游戏无缝转场 (Seamless Transition Flow)
* **单场景架构 (Single Scene Architecture)**：
  * 主菜单 (Menu) 与 游玩状态 (Gameplay) 共用同一真实游戏场景。
  * 处于主菜单时，左侧悬浮展示的并非虚假预制体，而是根据当前选中模式实时生成的真实塔数据模型。
* **待机交互 (Idle Interaction)**：
  * 模型处于待机状态时，会朝着随机方向缓慢自转。
  * 玩家可以用鼠标左键拖拽预览塔的局部结构，松手后模型恢复自转。
* **模式切换转场 (Mode Switching: Shatter & Reassemble)**：
  * 当玩家在右侧 UI 切换游戏模式时，左侧当前的塔瞬间失去引力约束，解体为无数独立方块（或沿核心轴向拉散）。
  * 方块在空中短暂悬浮/盘旋后，迅速向中心聚拢，并根据新模式的配置数据重组（Snap）成全新的塔模型。
* **进入游戏 (Enter Gameplay)**：
  * 玩家点击“开始游戏”后，右侧 UI 界面平滑移出视口。
  * 摄像机执行拉近（Zoom In）动画，模型自转立刻停止，并平滑旋转对齐至正确的解谜起始朝向视角。
* **退出游戏 (Exit to Menu)**：
  * 玩家在游玩中按下 `[ESC]` 或暂停菜单的“返回”，摄像机迅速拉远退回原展示机位。
  * 右侧 UI 重新划入，模型恢复原先的随机自转待机状态。

## 4. 设置项信息架构树 (Settings Information Architecture)
长列表从上至下依次排列：

### 4.1 【通用 General】
* **语言 (Language)**：下拉框（简体中文 / English）
* **屏幕震动 (Screen Shake)**：开关 Toggle
* **显示通关计时 (Show Timer)**：开关 Toggle
* **清除本地存档 (Clear Save Data)**：危险按钮（点击后弹出二次确认防呆弹窗）

### 4.2 【画面 Graphics】
* **全屏模式 (Fullscreen)**：开关 Toggle
* **画质预设 (Quality Preset)**：下拉框（低 / 中 / 高 / 极高）
* **抗锯齿 (Anti-Aliasing)**：开关 Toggle
* **泛光强度 (Bloom Intensity)**：滑动条 Slider

### 4.3 【控制 Controls】
* **鼠标灵敏度 (Mouse Sensitivity)**：滑动条 Slider
* **反转 X/Y 轴 (Invert X / Y)**：独立开关 Toggle
* **键位绑定表 (Keybindings)**：左侧为行为（如：向上旋转），右侧为当前绑定键（点击后进入“请按下按键...”的监听状态）。

### 4.4 【音频 Audio】
* **主音量 (Master Volume)**：滑动条 Slider (0-100%)
* **音乐音量 (Music Volume)**：滑动条 Slider
* **音效音量 (SFX Volume)**：滑动条 Slider
* **UI交互音量 (UI Volume)**：滑动条 Slider

## 5. 特殊机制：输入设备热插拔 (Input Hot-Swapping)
* **动态 UI 切换**：严禁在 UI 中混排键鼠和手柄设置。
* **事件监听**：底层 Input System 必须监听最后一次输入设备。
* **无缝替换**：若玩家操作了手柄摇杆或按键，全屏 UI 图标（如 `[Esc]` 变成 `[Options]`）以及设置内的【控制】模块内容，必须瞬间无缝替换为手柄专属设置（如摇杆死区、震动强度）。
