# Game Design Document: AR实体交互与具身认知

> **系统：** AR Tangible Interaction & Embodied Cognition  
> **版本：** 1.0  
> **状态：** 规划阶段（尚未实现）  
> **设计：** Game Designer

---

## 1. 设计概念

### 1.1 核心构想

玩家可使用**实体魔方零件**进行拼合，通过 AR 识别同步到数字游戏中。

这是本项目最具学术创新价值的特性，它在"纯虚拟解谜"和"物理操作"之间架设了一座桥梁。

### 1.2 设计目标

| 目标 | 说明 |
| :--- | :--- |
| 降低空间认知门槛 | 将抽象的"心理旋转"转化为具象的"物理旋转" |
| 提供认知脚手架 | 为空间焦虑的玩家提供物理辅助理解工具 |
| 提升交互沉浸感 | 手势操作增强大脑的空间处理能力 |
| 拔高学术价值 | 为毕业论文提供"具身认知"的创新实践支撑 |

---

## 2. 理论支撑：具身认知理论 (Embodied Cognition)

### 2.1 核心论点

> **认知不仅是大脑的活动，更是身体与环境互动的产物。**

- 涉及物理移动、手势操作或实体操作（如拼图、搭积木）的"动觉（kinesthetic）"参与，能**显著增强**大脑的空间处理和视觉化能力
- 物理操作为高抽象度任务提供了**认知脚手架（Cognitive Scaffolding）**

### 2.2 在本游戏中的意义

| 纯虚拟模式 | AR实体辅助模式 |
| :--- | :--- |
| 玩家需在脑中完成心理旋转 | 玩家可用手旋转物理魔方 |
| 高抽象度，认知负荷重 | 具象操作，认知负荷低 |
| 空间焦虑玩家容易受挫 | 物理操作缓解空间焦虑 |
| 纯数字反馈 | 触觉+视觉双通道反馈 |

### 2.3 学术论文中的定位

"具身认知"可作为论文的**差异化创新点**：
- 解释为什么 AR 实体交互能降低认知超载
- 提供"心理旋转"与"物理旋转"的对照实验设计基础
- 支撑"跨媒介交互降低空间焦虑"的学术论证

## 3. 技术设想

### 3.1 系统架构（规划）

```mermaid
graph LR
    A[实体魔方模型] -->|AR摄像头| B[AR终端]
    B -->|姿态识别| C[姿态数据]
    C -->|通信协议| D[Unity 运行时]
    D -->|状态同步| E[CubeModel 更新]
    E --> F[视图层渲染更新]
```

### 3.2 关键技术环节

| 环节 | 说明 |
| :--- | :--- |
| AR姿态捕获 | 识别实体魔方的面旋转状态 |
| 通信协议 | 实体操作实时同步到 Unity |
| 状态映射 | 物理魔方状态 → CubeModel 逻辑状态 |
| 冲突处理 | 物理操作与数字操作的状态一致性保障 |

### 3.3 DDA 集成

AR 实体交互可作为 DDA 的**辅助降级手段**：

| 触发条件 | 响应 |
| :--- | :--- |
| DDA 检测到深度焦虑 | 弹出提示：可使用实体魔方辅助思考 |
| 玩家连续撤回超过阈值 | 建议切换到 AR 辅助模式 |
| 停顿时间超过认知超载阈值 | 提供物理操作引导 |

## 4. 实体设计要求（规划）

### 4.1 魔方模型

| 参数 | 要求 |
| :--- | :--- |
| 结构 | 3×3×3 可旋转面 |
| 标识 | 每面/每块需有 AR 可识别的标记 |
| 材质 | 耐用、手感舒适 |
| 尺寸 | 适合单手操作 |

### 4.2 AR 识别标记

- 每个面需要独特的视觉标记（颜色/图案/AR码）
- 标记需在各种光照条件下可识别
- 旋转后的新面排列需要稳定识别
