# 原型实验目录

> **用途**：存放实验性设计文档和概念验证草案，与 GDD 主线完全隔离  
> **原则**：原型文档**不纳入** GDD 正式体系，不触发 Phase 4 文档同步  
> **毕业流程**：原型验证通过后，需通过 Phase 1 Proposal 正式提议"从原型毕业到主线"

---

## 原型状态追踪

| 原型名 | 创建日期 | 状态 | 关联系统 | 备注 |
|:---|:---|:---|:---|:---|
| （暂无活跃原型） | — | — | — | — |

**状态枚举**：
- 🔬 **实验中**：正在验证概念可行性
- ✅ **验证通过**：概念可行，待 Proposal 毕业
- ❌ **已放弃**：概念不可行或优先级变更
- 🎓 **已毕业**：已通过 Proposal 纳入主线

## 原型文档模板

```markdown
# 原型：[实验名]

> **创建日期**：YYYY-MM-DD  
> **状态**：🔬 实验中  
> **目标**：用一句话描述要验证什么  
> **关联系统**：涉及哪些现有系统

## 假设
- 假设 1：[需要验证的假设]
- 假设 2：...

## 方案草案
[快速草稿，不需要完整格式]

## 验证结果
- [ ] 假设 1：⬜ 未验证 / ✅ 通过 / ❌ 失败
- [ ] 假设 2：...

## 结论
[验证通过 → 下一步走 Proposal 毕业流程 / 失败 → 记录原因归档]
```
