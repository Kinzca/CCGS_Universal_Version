# 集中 Bug 追踪表 (Centralized Bug Tracker)

> **用途**：汇总所有 QA Report 和日常开发中发现的问题，提供全局可视的 Bug 状态一览。  
> **维护规则**：Phase 3 QA Report 中发现的问题必须同步登记到此表；Phase 0 上下文加载时扫描未关闭项。  
> **版本**：1.0

## 登记规范

### 新增 Bug 时

1. **分配 ID**：活跃问题使用 `BUG-XXX`（三位递增），关闭后迁移到"已关闭"表并改前缀为 `BUG-CXXX`
2. **标注严重度**：🔴 阻塞 / 🟡 一般 / 💭 建议
3. **标注模块**：生成 / 渲染 / 操控 / DDA / 配置 / 音频 / UI / 其他
4. **填写发现来源**：对应 QA Report 文件名或对话 ID
5. **填写发现日期**

### 关闭 Bug 时

1. 从"活跃问题"表移至"已关闭问题"表
2. 填写修复版本、关闭日期、修复说明
3. 更新关联 QA Report 中的对应问题状态
