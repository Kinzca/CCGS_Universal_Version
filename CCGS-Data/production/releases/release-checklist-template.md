# 发布检查清单模板 (Release Checklist)

> **用途**：每次版本发布前逐项检查，确保发布质量  
> **使用时机**：阶段完成时可选择性对照；正式发布前强制执行  
> **负责人**：开发者 + AI 协同检查

---

## 使用方法

1. 复制此模板为 `YYYY-MM-DD-[版本号]-release-checklist.md`
2. 逐项检查并标注状态
3. 所有 🔴 必须项通过后方可发布

---

## 1. 构建验证 🔴 必须

- [ ] 项目可正常编译，无编译错误
- [ ] 构建产物可正常启动
- [ ] 目标平台构建通过（Android / iOS / PC）
- [ ] 构建包大小在预算范围内
- [ ] 热更资源打包正常（YooAsset）

## 2. 性能基线 🔴 必须

- [ ] 目标设备帧率达标（≥ 30fps 最低 / ≥ 60fps 推荐）
- [ ] 内存占用在预算范围内
- [ ] 无明显卡顿和掉帧
- [ ] 加载时间可接受（首次启动 / 场景切换）
- [ ] 无内存泄漏（长时间运行测试）

## 3. QA 回归 🔴 必须

- [ ] Bug Tracker 中无 🔴 严重度未修复项
- [ ] 核心玩法循环可完整走通
- [ ] 所有已修复 Bug 的回归验证通过
- [ ] 边界情况测试通过（极端输入、快速操作）
- [ ] 崩溃率 ≤ 0.1%

## 4. 文档完备 🟡 建议

- [ ] GDD 文档与实际实现一致
- [ ] API 变更已在 Changelog 中记录
- [ ] 已知问题和限制已在发布说明中列出
- [ ] 用户手册/教程已更新（如有）

## 5. 平台合规 🟡 建议

- [ ] 应用商店截图和描述已更新
- [ ] 隐私政策符合目标平台要求
- [ ] 年龄分级评估完成
- [ ] 第三方 SDK 版本符合平台最新要求

## 6. 本地化 💭 可选

- [ ] 目标语言翻译完成
- [ ] 翻译后 UI 回归验证通过
- [ ] 文化敏感性审查通过
- [ ] 本地化就绪等级达到 L3
