# 回顾文档目录

此目录存放检查点 (Checkpoint) 回顾文档。

## 命名规范

`YYYY-MM-DD-[阶段]-[检查点]-retrospective.md`

示例：`2026-04-25-phase-c-cp0-retrospective.md`

## 触发条件

- 完成一个重要检查点 (CP) 时
- 完成一个完整开发阶段时
- 发生重大架构决策变更时

## 模板结构

每份回顾文档应包含：

1. **✅ 成功项** — 本阶段做得好的实践
2. **⚠️ 待改进** — 发现的问题和低效环节
3. **📝 行动项** — 具体可执行的改进措施
4. **🧠 教训沉淀** — 需要长期记住的经验（适合写入 roadmap §4）
