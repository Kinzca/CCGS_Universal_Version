## Bug Report
- **ID**: BUG-007
- **Title**: 弹性阻尼仅对横向拖拽(Y轴旋转)生效，竖向拖拽(X/Z轴旋转)无阻尼效果
- **Severity**: S2
- **Frequency**: Always
- **Build**: commit b688ba5
- **Platform**: Unity Editor
- **Status**: 🟢 Fixed (commit 9268196)

### Steps to Reproduce
1. 进入 Tower 场景，确认玩家角色站在某个方块上。
2. 左键点击侧面，**水平拖拽**玩家站立的层 → 观察到弹性阻尼效果（拉不动 + 快速回弹）✅ 正常。
3. 左键点击侧面，**垂直拖拽**玩家站立的列（外列）→ 观察效果。

### Expected Behavior
竖向拖拽到包含玩家站立位置的列时，同样应该触发弹性阻尼效果（阻尼感 + 松手回弹），与横向的体验一致。

### Actual Behavior
竖向拖拽时没有任何阻尼效果，直接按正常拖拽逻辑处理（可正常旋转并提交），保持了旧的行为。

### Root Cause Analysis
`IsRotationAllowed` 方法中只对 `RotateAxis.Y` 做了玩家站立层保护检查：

```csharp
if (axis == RotateAxis.Y)
{
    // 只检查了 Y 轴旋转...
}
return true; // X/Z 轴旋转永远返回 true
```

竖向拖拽映射到 `RotateAxis.X` 或 `RotateAxis.Z`（旋转侧面外列），此时方法直接返回 `true`，导致 `IsRubberBanding` 永远为 `false`。

### Suggested Fix
在 `IsRotationAllowed` 中增加对 X/Z 轴旋转的玩家站立列保护检查：
- 当 `axis == RotateAxis.X` 时，检查 `playerActor.StandCoord.x == layer`
- 当 `axis == RotateAxis.Z` 时，检查 `playerActor.StandCoord.z == layer`
