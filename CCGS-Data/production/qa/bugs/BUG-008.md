## Reproduction Steps
**Preconditions**: 进入游戏处于可以自由移动的状态。

1. 鼠标右键点击远处的地块目标点。
2. 观察玩家角色在开始移动瞬间的朝向。
3. 观察玩家角色在移动过程中的动画播放情况。

**Expected Result**: 
1. 玩家应该瞬间或平滑转向目标地块的方向。
2. 玩家在位移过程中必须播放走动/跑步的动画。

**Actual Result**: 
1. 部分情况下，玩家朝向与移动方向不一致。
2. 有时完全没有播放移动动画，角色以默认待机姿势（或上一个动作状态）平滑滑动到目标位置。

## Technical Context
- **Likely affected files**: `PlayerControllerSys.cs`, `PlayerState.cs`, `ClickToMoveSys.cs`
- **Related systems**: Input System (Right Click), Animation System, Pathfinding/NavMesh
- **Possible root cause**: 
  - **动画丢失/滑移原因**：连续寻路移动时，每一个单元格移动完毕后，`PlayerControllerSys.CompleteCurrentSegment` 会调用 `FinishMove()` 切回 `Idle` 状态。然后队列紧接着执行下一步，又调用 `StartMove()` 切入 `Moving` 状态。这样频繁的 `Idle -> Moving -> Idle` 会导致 Animator 状态机被反复重置，如果使用 CrossFade 或动画过渡时间较长，表现上就是一直在播第一帧（导致看起来没走路而是平移滑动）。
  - **朝向错误原因**：`PlayerControllerSys.StartMove` 和 `CompleteCurrentSegment` 中通过 `TryUpdateRotationForCurrentSegment()` 来转向。但在寻路转角处，如果由于某种原因移动前一瞬间还处于 `Idle` 且立刻被赋予下一步动作，或者 `TryProjectDirectionOnSupportPlane` 精度计算产生极小值导致 `return false`，转向就会失效。另外，在连续右键点击打断当前寻路时，旧队列的清除和新队列的赋值可能没有做好状态流转保护。

**Recommended Fix**: 
1. **动画**：在 `FinishMove()` 中不要立刻切回 `Idle`，可以引入短暂的延迟（或者通过 `PlayerMoveSys` 层面判断 Queue 是否还有剩余的 MoveRule），如果还有后续移动，就保持 `Moving` 状态。
2. **转向**：在 `PlayerControllerSys.ChangeState` 之外，增加强制在每一步起步时的转向插值（Slerp）。修改 `UpdateSegmentRotation` 为平滑旋转，而非瞬间赋值 `localRotation`。
