# 19_第十九章节

第19章从市场的角度去立项本章导读本章探讨的是如何根据市场去设立手游项目。论述的角度为品类市场调研、产品趋势预测、相关竞品分析、用户定位分析，市场空间分析，以尽可能全面清晰地认清局势，确立项目。

最终结论为我们可以先对某个游戏市场进行总体的市场调研，然后再对某个品类进行详尽的相关分析，最后核算可能的成本与收益确定我们的项目。

## 19.1 市场规模，类型占比情况首先找可以切入的领域，或者希望切入领域的市场分析数据报告，2018年手游市场的总规模为1339.6亿元人民币。

粗略按顶部前50的流水预估整体，竞速类游戏占5.8%，即竞速类游戏的市场规模可能在77.7亿元人民币。

## 19.2 产品趋势，IP占比情况游戏品类的发展趋势方向如下，一方面从MOBA到吃鸡，一方面从休闲到超休闲。

而IP对游戏的重要性也越来越大。IP是有作用的，但不是全能的。

假如你有一个知名IP，你可以根据知名IP自身的背景要素去设定自己游戏的题材与类型。但注意，有一些小IP拿了也没用，这是因为按照小IP做会让你的设计极大受限，并且获得的自然量也极为有限。

固然在谈渠道推广的时候，有IP的游戏会让渠道在初期给更好的推荐位，但最终还是看数据的，小IP在设计上对你造成的负面影响也许更大。当然，在你游戏做到一半，选择一个自由度高、修改量小的小IP则是一个可行的选择。

但IP定制项目则要非常慎重，也许用户极为小众，一些网站论坛的IP热度数据其实很虚，不一定能有多少可以转化为游戏用户。如果你与某个渠道关系很好，也可以在立项的时候重点考虑下他们的渠道用户属性与平台策略，做一个什么样的游戏会更加契合他们的需求。

当你根据上述综合选定自己要做的游戏类型甚至题材（有IP可以直接根据题材定IP，没IP最好进行产品分析后再定题材）后，你还需要进行竞品分析，下文中我就以跑酷产品为例。

## 19.3 相关竞品分析1．竞品分析的理论在你确定了需要分析的竞品后，要特别注意竞品分析的意义和要做的工作。

首先，利用市场调研大致确定了要做什么类型的游戏，那么现在你要分析竞品，更加地了解这群用户，选择一个可以切入的空间，并为之后制定详细的切入方案做准备。其次，当你对竞品有了大致的了解后，你就需要思考假如我要其中的某个类别，大概需要做到什么样的品质，大概需要多强的研发实力、研发周期、研发成本；这个领域就其市场情况是否值得切入。

最后，在你体验竞品的时候，简要地记录下操作模式、商业化方式、系统循环、游戏体验，以方便以后回顾分析对比。接下来就可以具体进行分析，并对某个领域划分细分流派了。

### 4．玩家满足选定了用户，获得了用户画像后，在用户方面的最后一步就是考虑如何去满足这群用户、获得竞品优势、让游戏在推出后目标用户愿意来玩。

这里有3个很容易走入的歧途。第一个是：自认为自己可以做出一个很创新的玩法来满足这部分玩家。

创新的制作是十分艰难的，你不一定做得出，或者时间与代价巨大。外加你看中的领域别人可能也看重，并且已经有几个同类竞品在做，就算你做得不一样，玩家也不一定会喜欢，也许他们更喜欢玩其他竞品甚至原品。

所以简单地想着我做得不一样，做创新，并没有在立项阶段解决问题，只是将问题丢给了制作阶段，要做好制作阶段花费巨大代价去解决的心理准备。第二个很容易走入的歧途是：玩家的用户调研结果会告诉你答案，做好用户调研就行了。

PS4的首席架构师马克·塞尔尼(Mark Cerny)曾说：有一个业界神话就是‘如果你想要游戏大卖，就要听玩家的’——我要说的是焦点测试（Focus Test/Focus Group，一种通过访谈的方式调查受众兴趣和偏好的市场调查方法）​，如果你想知道你应当在游戏中放入什么元素，或是想知道你应该做什么类型的游戏，你最不该做的事就是焦点测试。有几件事你可以通过焦点测试得到，即什么游戏最流行，什么在当下这个时间点最受欢迎——除非你的游戏在几周后就要发售了，否则测试的结果对你们的游戏来说是毫无意义的。

焦点测试者无法告诉你未来。马克·塞尔尼的观点只在一种情况下无效，那就是腾讯级的用户大数据，它可以借此在自己已运营的同类游戏用户中获得非常精细的定位与摸底，并有巨大量级做支撑，保证测试的随机性较低。

第三个很容易走入的歧途是：我直接买个IP，然后换个皮就行了。这个做法的问题在于，哪怕你买了一个大IP，借此低成本导入了很多用户，但这群用户与一般的游戏玩家是不同的，很容易与你目前的游戏玩法冲突。

例如，你买了个电视剧IP，那群电视剧休闲玩家可能根本就不喜欢战斗而只喜欢养成。又比如你买了个《拳皇》游戏IP，我们知道《拳皇》在街机上的操作频率超高，但在手机上因为操作模式，可能他们完全无法接受。

就算你买个电脑端游戏IP，电脑端较长时间的玩法可能与手游碎片化的玩法不兼容。而这个问题最好的解决方法还是针对目前的原型游戏去思考的，它不能改成什么样，然后考虑它无法改动的方面，你去突破，是否可以满足足够量级的玩家。

当然，原型游戏也可以换皮重做一个游戏，但这就是另外一个问题了，这个问题我也会在后文中另外论述。

## 19.5 市场空间分析1．推广买量成本我们通过上面的分析，已经大致可知在什么国家要做，针对什么年龄层用户的游戏，假如有好几个备选方案，则全部分析一遍。

那么针对这个年龄层，常用的推广方式和平台是什么，成本又是多少，你用什么样的方式将用户导入，怎么样用最低的成本获得最优质的客户。这里应该对用户导入成本和收益进行计算，由于本书暂时没有很确切的跑酷类游戏不同包体大小和题材的全面买量成本数据（或不走买量、走联运平台、应用商店、KOL、直播等方式）​，先简单假设包体越大，买量成本越高。

### 2．付费与成本分析在考虑了买量成本之后，我们还要简单地考虑我们可以在该游戏上赚多少钱。

假如你可以直接拿到该游戏的细化付费数据，那么就可以很好地进行估算。假如你认为对方的付费有优化的空间，你就需要分析相关产品的付费行为是什么样，它们是通过什么付费点刺激玩家付费的；分析他们的定价策略和活动。

然后找到较为充分的依据，认定自己一定可以做得更高。有关对方的付费点，可以在Appannine软件上查询到一部分。

虽然没有明确的购买数量，但至少你知道了购买的频率排序，其可以帮助你推测玩家在某方面的付费力度。有了买量成本和预估的玩家付费数值模型，我们接着计算导入一个用户大概需要多少钱，每个用户你大概可以赚多少钱，研发这款游戏的成本又是多少，最终需要生命周期总价值(Life Time Value，LTV)＞每行动成本(Cost Per Action，CPA)。

LTV是指一个用户离开游戏前平均会花多少钱。如每个玩家平均玩3个月，每个月花60元，那么一个用户的LTV就是180。

CPA是指用户获取成本，如每次点击广告将花费1元，每次100次点击产生一个进入游戏的玩家，那么每个玩家的CPA就是100元。具体的算法会和LTV的算法相关，其结果就是你会知道每个用户让他进入你游戏花多少钱、每买进一个用户会净赚多少钱。

如果你不是研运一体，只是做纯研发的，还要扣除给发行的分成和各种税，最后拿到手的才是净收入。这个环节的意义是通过LTV＞CPA，综合对比往哪个市场、哪个方向去做可以获得更大的收益。

同时还要考虑各种成本，包括但不限于游戏开发成本、游戏运维成本、IP成本等，根据净收入计算回本周期，评估游戏是否可以高收入运营这么长时间。事实上还要做收入衰减模型，由于这里面有太多细节，本文只简单概括，介绍下思路。

最终确定这个可以做、值得做，然后才开始做具体的立项设计。相当于种花，这决定了你以什么样的形式种什么样的花，好比我到底是野生放养的种牡丹，还是在温室大棚中种玫瑰。

### 3．未来规划艰难以上都是最简单的举例，不保证跑酷游戏方面结论的正确，只保证其方法的通用性。

我们行业内习惯用Come（怎么让用户进来）​、Stay（怎么让用户留下）​、Pay（怎么让用户花钱）去解释立项的核心思路，但其实立项最难的并不是这3大块。在立项分析时，整件事最难的是对未来的判断，现在一般做一个游戏要1～2年（小游戏除外）​，那么在1～2年后，市场会变得如何、玩家想要玩什么样的游戏、他们的用户属性是不是发生了变化、其他竞品又会对我们造成什么影响？

法国战略学者博弗尔在其著作《战略绪论》中曾这么写道：​“我可以做一个并不夸张的简单比喻，战略家就好像是一位外科医生，被请去给一个病人做手术，那个病人的身体正在继续不断地以极高的速度成长着，所以他对于病人的详细解剖部位无法确定、而他的手术台也处于不断运动的状态下，同时他还必须在现在就订购五年后才使用的仪器。​”同样，我们在立项时必须把握这些不确定性的要素，试图在未来获得优势。

但比把握变化更好的方法，其实是把握那些不变，无论做任何项目，我们都需要高效的组织团队成员去做，尤其是创新的项目，对研发方式就更是讲究，所以接下来要说的是高效地做创新。

## 19.6 本章技巧概述技巧78 市场类立项的校验流程与破除幻象在正文中，我写了先进行手游市场总体的市场调研，选定一个品类后（如跑酷）再进行市面上相关产品的详尽分析，试图从中选定一个有市场空间的赛道，去评估方向是否可行，不过在其中也有很多细节的点，会阻碍我们从市场去立项。

首先我们必须认识到，游戏不是APP。用户没有需求，我们是在创造需求。

如果说用户喜欢玩游戏，那么他的本质就是需要娱乐想要休闲，他不能刷短视频吗？好玩不是本质，体验的独特性才是。

基于独特性来说，新奇玩法的移植可以产生，利用技巧69与70，进行巧妙的题材切换也可以产生。正文是数据型分析模拟，我们也可以实际测试模拟，我们可以用投放一个游戏的概念视频广告，看用户对其的点击率，判断吸量程度。

我们可以用好几个概念视频去确认吸量程度，最终选出我们觉得最好的那一个。此时我们又需要思考，游戏的题材/玩法/宣传概念，所能吸引到的用户量级是否足够大，这个玩法是否可以做高额付费，这个题材/美术风格是否需要高额成本。

根据这个问题的答案，我们可以接着思考。在此条件下，该游戏的竞品优势大不大，商业模式是快速回收还是长线运营，是自己买量运营还是找发行代理推广。

自己所设定的产品条件，对外界的竞争力，吸引力是否足够，在此情况下推广与买量成本大概会是多少，在此情况下别人是否愿意推，为了增加产品的竞争力，是否有必要加大产品的美术投入，或者公司是否能研运一体的自己做。基于以上思考研究，我们就可以破除一些幻想。

第一层幻象是技术崇拜，有一些专家会觉得这个玩法/画风/技术领先行业，一定可以成。这个真不一定，它可能被其他短板给拖死。

第二层幻象是产品崇拜，觉得酒香不怕巷子深，我只要全力用心地做好产品，就一定可以成功。这个也不一定，游戏好但不赚钱，没广告给你推广，玩的人怎么进来呢？

第三层幻象是商品崇拜，觉得流水可以起来，能赚钱的商品就一定是好商品。那也要看收益率，游戏上线后要收回研发资金，还有版本更新和运营维护成本，如果研发成本3000万，研发最终到手月入100万，但项目月固定开支就是95万，考虑到流水还会衰退，那公司还有继续做的动力吗？

研发有风险，如果收益率很低，是不是还不如存银行呢？所以基于市场立项，就一定要去校验，市场上接受什么，我们去制作它，应该用什么样的成本和方式最恰当，这么做了以后，成本与收益是否对等，是否值得去冒险立项并研发。

但即使我们想得再好，校验得很充分，还是有可能在上线后才发现不是那么回事。以上说的都是只是通用的做法，但游戏领域与种类的不同，导致了做法也完全不同，游戏的类型太多了，每一个局部都有自己的特殊情况。

有的领域看着有空间，其实根本不可能进入，有的关联了渠道，有的基础环境将发生变化，有的有特殊优势。加上买量价格不断上涨，玩家对游戏也越来越挑剔，把做游戏看成市场与流量的生意，去洗量已经越来越难。

我们还是要尽量避免投机，基于自身优势、积累、热爱出发，去更沉稳地做游戏，甚至做创新。
