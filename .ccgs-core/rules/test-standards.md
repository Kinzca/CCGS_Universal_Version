# 测试标准规则 (Test Standards Rules)

> **适用路径**: `tests/`, 以及所有测试相关文件
> **原 CCGS 对应**: `.claude/rules/test-standards.md`

- 测试命名：`[系统]_[功能]_Test.cs`（文件）；`Test_[场景]_[预期结果]`（方法）
- **确定性**：每次运行必须产生相同结果——禁止随机种子、禁止时间依赖断言
- **隔离性**：每个测试独立设置和清理状态；测试间禁止依赖执行顺序
- **无硬编码数据**：测试夹具使用常量文件或工厂方法，不要内联魔法数字
  （例外：边界值测试中精确数字本身就是测试要点）
- **独立性**：单元测试不调用外部 API、数据库或文件 I/O——使用依赖注入
- 核心算法（生成器、求解器、DDA）测试覆盖率目标 **100%**
- 自动测试套件在每次 push 到 main 和每个 PR 时运行
- **禁止**跳过失败测试来使 CI 通过——修复根本问题

## 交错空间必测项

| 系统 | 必测场景 |
|:---|:---|
| 塔窗生成 | 可解性验证、打乱步数 M 正确性 |
| DDA | CSR ≈ 1.04 参数调控 |
| 续关 | 入口一致性、NextEntrySpec 正确性 |
| 配置表 | Luban 加载完整性 |
